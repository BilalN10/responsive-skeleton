Additional information

appModel = ApplicationDataModel

Comme dit précédement, appModel va contenir toutes nos configurations, de Front-End jusqu'à Firebase / Stripe.

Main Application Data

2 champs sont obligatoires pour la configuration:

final appModel = ApplicationDataModel(
    appLogo: "svg/example_logo.svg", // path vers le logo (/!\ fichier SVG uniquement)

    appTitle: "Example App", // Nom de l'application (équivalent de la balise <title> en HTML)
);

Firebase

final appModel = ApplicationDataModel(
    ...,
    firebaseIsEnabled: true, // true par défaut
    
    firebaseOptions: const FirebaseOptions(...), // objet Firebase options 
    // contenant tous les identifians nécéssaires à la configuration / connexion Firebase.
    
    userCollection: "users/", // Nom de la collection où les utilisateurs sont stockés
    // "users/" par défaut.

    firebaseMessagingIsEnabled: true, // true par défaut, Si FirebaseMessaging est utilisé 
    // dans le projet
);

Responsive

Les champs ci dessous permettent de définir les breakpoints au builder Responsive.

final appModel = ApplicationDataModel(
    ...,
    maxPhoneWidth: 800, // 800 par défaut

    maxTabletWidth: 1200, // 1200 par défaut

    maxWebWidth: 2400, // 2400 par défaut

    maxWidth: double.infinity, // double.infinity par défaut
    // sert pour la taille décran maximum. Si jamais la valeur n'est pas double.infinity et
    // l'écran est plus large, vous aurez 1 bande blanche (par défaut) de chaque côté.
);

Autres

Plusieurs autres champs de configurations sont disponibles dans le ApplicationDataModel, n'hésitez pas à regarder le code, celui-ci est aussi documenter.
